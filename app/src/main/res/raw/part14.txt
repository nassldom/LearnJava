<b>Part 14: Linked Lists (Kapitel 130-133)</b>

In diesem Teil lernst du die Konzepte von verketteten Listen (Linked Lists) kennen – eine dynamische Datenstruktur, die im Gegensatz zu Arrays flexibel wachsen kann.

<b>1. Das Node-Konzept (Kapitel 130)</b>
Eine Linked List besteht aus einzelnen **Knoten** (Nodes). Jeder Knoten speichert Daten und einen Verweis auf den **nächsten** Knoten.
- <code>Head:</code> Ein Verweis auf den ersten Knoten der Liste. Ist dieser <code>null</code>, ist die Liste leer.
[CODE]
public class Node {
    int data;
    Node next;

    Node(int data) {
        this.data = data;
        this.next = null;
    }
}
[/CODE]

<b>2. Listen-Operationen (Kapitel 131-132)</b>
Das Arbeiten mit Listen erfordert das "Traversieren" (Durchlaufen) der Knotenkette.
- <code>Einfügen:</code> Neue Knoten werden meist am Anfang oder Ende angehängt.
- <code>Traversieren:</code> Man startet beim <code>head</code> und folgt den <code>next</code>-Verweisen in einer Schleife.
[CODE]
Node current = head;
while (current != null) {
    System.out.println(current.data);
    current = current.next;
}
[/CODE]

<b>3. Generische Listen (Kapitel 133)</b>
Durch **Generics** kann eine Linked List so programmiert werden, dass sie jeden beliebigen Datentyp (z.B. <code>String</code>, <code>Integer</code>, eigene Objekte) speichern kann.
[CODE]
public class GenericNode<E> {
    E data;
    GenericNode<E> next;
    // ...
}

// Nutzung:
GenericLinkedList<String> namen = new GenericLinkedList<>();
[/CODE]

<b>Merksatz:</b>
Während Arrays schnellen Zugriff per Index bieten, sind Linked Lists effizienter beim häufigen Einfügen und Löschen von Elementen, da keine anderen Elemente verschoben werden müssen.
