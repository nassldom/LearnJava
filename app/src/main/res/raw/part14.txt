<b>Part 14: Linked Lists (Kapitel 130-133)</b>

In diesem Teil lernst du die Konzepte von verketteten Listen (Linked Lists) kennen – eine dynamische Datenstruktur, die im Gegensatz zu Arrays flexibel wachsen kann.

<b>1. Das Node-Konzept (Kapitel 130)</b>
Eine Linked List besteht aus einzelnen <b>Knoten</b> (Nodes). Jeder Knoten speichert Daten und einen Verweis auf den <b>nächsten</b> Knoten.
- <code>Head:</code> Ein Verweis auf den ersten Knoten der Liste. Ist dieser <code>null</code>, ist die Liste leer.
[CODE]
public class Node {
    int data;
    Node next;

    Node(int data) {
        this.data = data;
        this.next = null;
    }
}
[/CODE]

<b>2. Listen-Operationen (Kapitel 131-132)</b>
Man unterscheidet zwischen unsortierten (unordered) und sortierten (ordered) Listen.
- <code>Einfügen:</code> Bei unsortierten Listen meist am Anfang (sehr effizient). Bei sortierten Listen wird die passende Stelle gesucht.
- <code>Löschen:</code> Ein Element wird entfernt, indem der <code>next</code>-Verweis des Vorgängers auf den Nachfolger gesetzt wird.
- <code>Traversieren:</code> Das Durchlaufen der Liste von <code>head</code> bis zum Ende.
[CODE]
Node current = head;
while (current != null) {
    System.out.println(current.data);
    current = current.next;
}
[/CODE]

<b>3. Generische Listen (Kapitel 133)</b>
Durch <b>Generics</b> kann eine Linked List so programmiert werden, dass sie jeden beliebigen Datentyp (z.B. <code>String</code>, <code>Integer</code>) speichern kann.
[CODE]
public class GenericNode<E> {
    E data;
    GenericNode<E> next;
    // ...
}

// Nutzung:
GenericLinkedList<String> list = new GenericLinkedList<>();
[/CODE]

<b>Merksatz:</b>
Während Arrays schnellen Zugriff per Index bieten, sind Linked Lists effizienter beim häufigen Einfügen und Löschen von Elementen, da keine anderen Elemente verschoben werden müssen.
