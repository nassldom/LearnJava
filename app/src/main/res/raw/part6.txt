<b>Part 6: Objektorientierte Programmierung (Kapitel 40-54)</b>

In diesem Teil lernen wir das Herzstück von Java kennen: Die Objektorientierte Programmierung (OOP).

<b>1. Klassen und Objekte (Kapitel 40)</b>
Eine <b>Klasse</b> ist ein Bauplan (z.B. der Bauplan eines Autos). Ein <b>Objekt</b> ist das konkrete Ding, das nach diesem Plan gebaut wurde (z.B. dein roter VW Golf).

[CODE]
String text = new String("Hallo"); // "text" ist ein Objekt der Klasse String
[/CODE]

<b>2. Referenzen (Kapitel 41)</b>
Variablen für Objekte speichern nicht das Objekt selbst, sondern eine <b>Referenz</b> (die Speicheradresse). Wenn du sagst <i>Auto meinAuto = deinAuto;</i>, zeigen beide Variablen auf dasselbe Auto!

<b>3. Eigene Klassen definieren (Kapitel 46)</b>
Du kannst eigene Baupläne schreiben. Diese enthalten <b>Attribute</b> (Daten) und <b>Methoden</b> (Verhalten).

[CODE]
class Hund {
    String name;
    int alter;

    // Konstruktor: Initialisiert das Objekt
    Hund(String name, int alter) {
        this.name = name;
        this.alter = alter;
    }

    void bellen() {
        System.out.println(name + " sagt: Wuff!");
    }
}
[/CODE]

<b>4. Kapselung & Sichtbarkeit (Kapitel 50)</b>
Um Daten zu schützen, verwenden wir <i>private</i>. Nur Methoden der eigenen Klasse können darauf zugreifen. Von außen nutzt man "Getter" und "Setter".
- <b>public:</b> Jeder darf zugreifen.
- <b>private:</b> Nur die eigene Klasse darf zugreifen.

<b>5. Statische Methoden & Variablen (Kapitel 54)</b>
Das Schlüsselwort <i>static</i> bedeutet, dass etwas zur <b>Klasse</b> gehört und nicht zu einem einzelnen Objekt.
Beispiel: <i>Math.sqrt()</i> ist statisch. Du musst kein Objekt der Klasse "Math" erstellen, um es zu nutzen.

[CODE]
public static void main(String[] args) {
    // Statische Methoden werden direkt über den Klassennamen aufgerufen
}
[/CODE]
