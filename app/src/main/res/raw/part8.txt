<b>Part 8: Datei-Ein- und Ausgabe (Kapitel 70-73)</b>

In diesem Teil lernst du, wie man Daten dauerhaft in Textdateien speichert und wieder ausliest.

<b>1. Dateiumleitung (Kapitel 70-71)</b>
Anstatt Eingaben mühsam per Tastatur zu machen oder Ausgaben nur in der Konsole zu sehen, kann man das Betriebssystem nutzen, um Daten aus Dateien zu lesen oder in sie zu schreiben.
- <b><:</b> Liest Eingaben aus einer Datei (<code>java Programm < input.txt</code>).
- <b>>:</b> Schreibt Ausgaben in eine Datei (<code>java Programm > output.txt</code>).

<b>2. In Dateien schreiben mit PrintWriter (Kapitel 72)</b>
Um Text direkt aus Java in eine Datei zu schreiben, nutzen wir die Klasse <code>PrintWriter</code>.
[CODE]
PrintWriter ausgabe = new PrintWriter(\"ergebnis.txt\");
ausgabe.println(\"Das wird in der Datei gespeichert.\");
ausgabe.close(); // WICHTIG: Datei immer schließen!
[/CODE]

<b>3. Aus Dateien lesen mit Scanner (Kapitel 72)</b>
Das Lesen funktioniert fast wie die Eingabe über die Tastatur, nur dass wir ein <code>File</code>-Objekt übergeben.
[CODE]
Scanner scan = new Scanner(new File(\"daten.txt\"));
while (scan.hasNextLine()) {
    String zeile = scan.nextLine();
    System.out.println(zeile);
}
scan.close();
[/CODE]

<b>4. Wichtige Techniken beim Lesen (Kapitel 73)</b>
Es gibt verschiedene Strategien, um das Ende einer Datei zu erkennen:
- <b>Zählschleife:</b> Die erste Zahl in der Datei gibt an, wie viele Werte folgen.
- <b>Sentinel-Wert:</b> Ein spezieller Wert (z.B. -1) markiert das Ende der Daten.
- <b>hasNext-Prüfung:</b> Die Schleife läuft so lange, wie noch Daten vorhanden sind (<code>hasNextInt()</code>, <code>hasNextDouble()</code>).

<b>Merksatz:</b>
Vergiss niemals <code>scan.close()</code> oder <code>ausgabe.close()</code>! Ohne diesen Befehl werden die Daten oft nicht physisch auf die Festplatte geschrieben und der Speicher bleibt belegt. Achte zudem auf <code>IOException</code>, die beim Dateizugriff auftreten können.
