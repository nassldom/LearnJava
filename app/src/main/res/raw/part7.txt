Teil 7: Arrays

Kapitel 60 – Arrays (Felder)

Ein Array ist eine geordnete Liste von Werten desselben Datentyps mit fester Größe. Statt 100 einzelne Variablen zu deklarieren, reicht ein Array. Der Index beginnt immer bei 0. Arrays sind Objekte – sie werden mit new erstellt.
Deklaration und Konstruktion:

[CODE]
// Deklaration + Konstruktion (Zellen mit 0 initialisiert)
int[] stuff = new int[5];

stuff[0] = 23;
stuff[1] = 38;
stuff[2] = 7 * 2;

System.out.println("stuff[0] has " + stuff[0]); // 23
System.out.println("stuff[3] has " + stuff[3]); // 0 (Standardwert)
Index als Variable nutzen:
java
double[] val = new double[4]; // alle Zellen mit 0.0 initialisiert
val[0] = 0.12;
val[1] = 1.43;
val[2] = 2.98;

int j = 3;
System.out.println("cell 3: " + val[j  ]); // 0.0
System.out.println("cell 2: " + val[j-1]); // 2.98
j = j - 2;
System.out.println("cell 1: " + val[j  ]); // 1.43
[/CODE]

Der Index darf ein beliebiger int-Ausdruck sein. Wird auf einen Index außerhalb der Grenzen zugegriffen, wirft Java automatisch eine ArrayIndexOutOfBoundsException – das ist die automatische Bereichsprüfung.

Initialisiererlisten – Array direkt beim Erstellen befüllen:

[CODE]
int[]    valA = { 12, 23, 45, 56 }; // Größe automatisch = 4
double[] valB = new double[6];

valB[0] = valA[2]; // 45
valB[3] = valA[2]; // 45
valB[1] = valA[1]; // 23
valB[5] = valA[0]; // 12
[/CODE]

Merke: Arrays haben eine feste Größe – einmal erstellt, kann sie nicht geändert werden. array.length (kein ()) gibt die Anzahl der Elemente zurück.




Kapitel 61 – Typische Array-Algorithmen

Dieses Kapitel zeigt die häufigsten Operationen auf Arrays: mit Schleifen durchlaufen, Maximum suchen, Summe und Durchschnitt berechnen.
Klassische und enhanced for-Schleife:

[CODE]
int[] zahlen = { 5, 3, 8, 1, 9 };

// Klassisch – Zugriff auf Index möglich:
for (int i = 0; i < zahlen.length; i++)
    System.out.println(zahlen[i]);

// Enhanced for (for-each) – kürzer, kein Index:
for (int wert : zahlen)
    System.out.println(wert);

Maximum finden – aus ArrayDemo.java:

class ArrayOps {
    // Überladung für int[]
    public static int findMax(int[] x) {
        int max = x[0]; // mit erstem Element starten, nicht mit 0!
        for (int index = 0; index < x.length; index++)
            if (x[index] > max)
                max = x[index];
        return max;
    }

    // Überladung für double[]
    public static double findMax(double[] x) {
        double max = x[0];
        for (int index = 0; index < x.length; index++)
            if (x[index] > max)
                max = x[index];
        return max;
    }
}

int[]    arI = { -20, 19, 1, 5, -1, 27, 19, 5 };
double[] arF = { 2.1, -4.0, 13.2, 21.95, -6.3, 3.28, 6.0, 0.5 };

System.out.println(ArrayOps.findMax(arI)); // 27
System.out.println(ArrayOps.findMax(arF)); // 21.95

Summe und Durchschnitt:

int[] werte = { 10, 20, 30, 40, 50 };
int summe = 0;

for (int w : werte)
    summe += w;

double durchschnitt = (double) summe / werte.length;
System.out.println("Summe: " + summe);            // 150
System.out.println("Durchschnitt: " + durchschnitt); // 30.0
[/CODE]

Merke: Maximum immer mit x[0] initialisieren – nicht mit 0, da das Array negative Werte enthalten kann. Die enhanced for-Schleife ist ideal fürs Durchlaufen, wenn kein Index benötigt wird.






Kapitel 62 – Objekte, die Arrays enthalten

Arrays sind Objekte in Java – eine Arrayreferenz kann deshalb als Instanzvariable in einer Klasse gespeichert werden. Damit lassen sich Klassen bauen, die intern Listen von Daten verwalten.
InputArray.java – Array per Nutzereingabe befüllen und ausgeben:

[CODE]
class InputArray {
    public static void main(String[] args) {
        int[] array = new int[5];
        int   data;
        Scanner scan = new Scanner(System.in);

        // Array befüllen
        for (int index = 0; index < array.length; index++) {
            System.out.println("enter an integer: ");
            data = scan.nextInt();
            array[index] = data;
        }

        // Array ausgeben
        for (int index = 0; index < array.length; index++)
            System.out.println("array[" + index + "] = " + array[index]);

        scan.close();
    }
}

Array als Instanzvariable in einer Klasse:

public class Notenliste {
    private int[]  noten;
    private String kursname;

    public Notenliste(String name, int anzahl) {
        kursname = name;
        noten = new int[anzahl]; // Array wird im Konstruktor erstellt
    }

    public void setNote(int index, int note) { noten[index] = note; }
    public int  getNote(int index)           { return noten[index]; }

    public double durchschnitt() {
        int summe = 0;
        for (int n : noten) summe += n;
        return (double) summe / noten.length;
    }
}

Boolean-Arrays – auch boolean[] ist möglich (Standardwert: false):

boolean[] flags = new boolean[5];
flags[2] = true;
// flags: false, false, true, false, false
[/CODE]

Merke: Die Instanzvariable speichert nur die Referenz auf das Array-Objekt. Das eigentliche Array wird mit new im Konstruktor erstellt. Ohne new hat die Variable den Wert null.


Kapitel 63 – Arrays als Parameter

Arrays werden als Referenz an Methoden übergeben – die Methode arbeitet direkt mit dem originalen Array, nicht mit einer Kopie. Änderungen innerhalb der Methode wirken sich dauerhaft auf das Array aus.

ChangeTest.java – Array-Parameter in Methoden:

[CODE]
class ChangeArray {

    // Array ausgeben – nur lesend
    public void print(int[] x) {
        for (int j = 0; j < x.length; j++)
            System.out.print(x[j] + " ");
        System.out.println();
    }

    // Array kopieren – source lesen, target verändern
    public void copy(int[] source, int[] target) {
        for (int count = 0; count < source.length; count++)
            target[count] = source[count];
    }
}

public class ChangeTest {
    public static void main(String[] args) {
        ChangeArray cng = new ChangeArray();
        int[] s = { 27, 19, 34, 5, 12 };
        int[] t = new int[s.length];

        cng.copy(s, t);
        System.out.println("After copy:");
        cng.print(t); // 27 19 34 5 12
    }
}

Wichtiger Unterschied zu primitiven Typen:

// Primitiv: Kopie übergeben – Original bleibt unverändert
public static void verdoppeln(int x) { x = x * 2; } // wirkungslos!

// Array: Referenz übergeben – Original wird geändert!
public static void verdoppeln(int[] arr) {
    for (int i = 0; i < arr.length; i++)
        arr[i] = arr[i] * 2; // Original wird verändert
}
Statische Methode mit Array-Parameter – nur lesend:

public static int summe(int[] arr) {
    int sum = 0;
    for (int wert : arr) sum += wert;
    return sum; // Array bleibt unverändert
}
[/CODE]

Merke: Arrays als Parameter ermöglichen wiederverwendbare Methoden ohne Datenkopie – effizient und schnell. Soll das Original nicht verändert werden, muss die Methode nur lesend auf das Array zugreifen.



Kapitel 64 – Methoden, die Arrays verändern

1. Call by Value (Wertübergabe)

Java verwendet ausschließlich Call by Value bei der Parameterübergabe. Das bedeutet: Der Inhalt einer Variable wird in den formalen Parameter kopiert.
Beispiel mit primitiver Variable:

[CODE]
class Alteration {
    void zero(int x) {
        x = 0; // ändert nur die lokale Kopie!
    }
}
[/CODE]

Wird alt.zero(value) aufgerufen, wird nur der Wert von value (z.B. 27) in x kopiert. Die Variable value in main() bleibt unverändert (After: 27).
Merke: Eine Methode kann eine primitive Variable des Aufrufers nicht ändern.


2. Arrays als Parameter – der Unterschied

Arrays sind Objekte. Beim Übergeben eines Arrays als Parameter wird eine Referenz (Zeiger auf das Array-Objekt) kopiert – nicht das Array selbst.
Das bedeutet: Die Methode hat direkten Zugriff auf das Array-Objekt und kann dessen Elemente verändern.

Beispiel:

[CODE]
class ChangeArray {
    public void zeroElt(int[] x, int elt) {
        if (elt < x.length)
            x[elt] = 0; // ändert das ECHTE Array!
    }
}
[/CODE]

Ausgabe nach cng.zeroElt(value, 0):

Before: 27 19 34 5 12
After:  0  19 34 5 12

3. Methode zum Nullsetzen aller Elemente (zeroAll)
[CODE]
// Alle Elemente auf 0 setzen
public void zeroAll(int[] ar) {
    for (int j = 0; j < ar.length; j++)
        ar[j] = 0;
}
[/CODE]
Wichtig: ar.length verwenden, niemals eine feste Zahl wie 5 – so funktioniert die Methode für jedes int-Array.

4. Lokale Variablen gehören nur ihrer Methode

Jede Methode hat ihre eigenen lokalen Variablen. Eine Methode kann nicht die lokale Variable einer anderen Methode benutzen.

Fehlerbeispiel:

[CODE]
public void zeroAll(int[] ar) {
    for (j = 0; j < ar.length; j++) // FEHLER: j ist nicht deklariert!
        ar[j] = 0;
}
Das j aus der print()-Methode ist unsichtbar in zeroAll(). Der Compiler meldet sofort einen Syntaxfehler.

5. Array-Kopier-Methode (copy)

Beide Arrays müssen vorher existieren und die gleiche Länge haben.

[CODE]
// Copy source to target
public void copy(int[] source, int[] target) {
    for (int count = 0; count < source.length; count++)
        target[count] = source[count];
}
Verwendung in main():
java
int[] s = {27, 19, 34, 5, 12};
int[] t = new int[s.length]; // Ziel-Array erstellen!
cng.copy(s, t);
Ausgabe:
text
After copy: 27 19 34 5 12
Achtung – falsche Version:
java
void copy(int[] source, int[] target) {
    target = source; // FALSCH! Ändert nur den lokalen Parameter
}
[/CODE]
Diese Version funktioniert nicht: target ist ein formaler Parameter von copy(). Die Zuweisung target = source ändert nur die lokale Referenz, nicht das echte Array t in main().

Zusammenfassung der Kernregeln

Parametertyp		Was wird übergeben?		Kann Methode den Wert im Aufrufer ändern?

int (primitiv)		Kopie des Wertes		Nein
int[] (Array/Objekt)	Kopie der Referenz		Ja (Elemente können geändert werden)


Vollständiges Beispielprogramm (ChangeTest.java)

[CODE]
// Array Example
class ChangeArray {
    public void print(int[] x) {
        for (int j = 0; j < x.length; j++)
            System.out.print(x[j] + " ");
        System.out.println();
    }

    // Copy source to target
    public void copy(int[] source, int[] target) {
        for (int count = 0; count < source.length; count++)
            target[count] = source[count];
    }
}

public class ChangeTest {
    public static void main(String[] args) {
        ChangeArray cng = new ChangeArray();
        int[] s = {27, 19, 34, 5, 12};
        int[] t = new int[s.length];

        cng.copy(s, t);
        System.out.println("After copy:");
        cng.print(t);
    }
}
[/CODE]


Kapitel 65 – Arrays von Objekten

1. Wiederholung: Objekt-Referenzen

Eine Variable wie String str; deklariert nur eine Referenzvariable – es wird noch kein Objekt erstellt. Ein Objekt entsteht erst durch Konstruktion:

String str;               // deklariert eine Referenzvariable
str = "Hello World";      // erstellt das Objekt und speichert die Referenz

2. Array von String-Referenzen – Schritt für Schritt

Schritt 1: Referenzvariable für ein Array deklarieren:

String[] strArray;              // 1. noch kein Array-Objekt
Schritt 2: Das Array-Objekt erstellen (z.B. 8 Zellen):

strArray = new String[8];       // 2. Array mit 8 leeren Zellen (alle = null)
Schritt 3: Einzelne Zellen befüllen:

strArray[0] = "Hello";          // 3. Referenz in Zelle 0 speichern
strArray[1] = "World";
Oder kombiniert in einer Zeile:

String[] strArray = new String[8];
Wichtig: Alle leeren Zellen eines Objekt-Arrays werden automatisch mit null initialisiert ("kein Objekt").

3. Länge und häufiger Fehler

strArray.length = Anzahl der Zellen (hier: 8), nicht die Anzahl der befüllten Zellen!

Letztes Element ansprechen:

strArray[strArray.length - 1] = "the end";

Jede String-Referenz im Array kann auf einen String unterschiedlicher Länge zeigen – die Strings müssen nicht gleich lang sein.

4. Strings ausgeben & null vermeiden
[CODE]
for (int j = 0; j < strArray.length; j++) {
    if (strArray[j] != null)
        System.out.println("cell " + j + ": " + strArray[j]);
    else
        System.out.println("cell " + j + ": empty");
}
[/CODE]

Achtung: Niemals strArray[j].length() aufrufen, wenn strArray[j] möglicherweise null ist – das führt zu einem Laufzeitfehler!

5. Vollständiges lauffähiges Beispiel

[CODE]
// StringArray.java
public class StringArray {
    public static void main(String[] args) {
        String[] strArray = new String[8];

        strArray[0] = "Hello";
        strArray[1] = "World";
        strArray[2] = "Greetings";
        strArray[3] = "Jupiter";
        strArray[strArray.length - 1] = "the end";

        for (int j = 0; j < strArray.length; j++) {
            if (strArray[j] != null)
                System.out.println("cell " + j + ": " + strArray[j]);
            else
                System.out.println("cell " + j + ": empty");
        }
    }
}
[/CODE]

Ausgabe:

cell 0: Hello
cell 1: World
cell 2: Greetings
cell 3: Jupiter
cell 4: empty
cell 5: empty
cell 6: empty
cell 7: the end

6. Kommandozeilenargumente (String[] args)

Die Signatur public static void main(String[] args) ist dir bekannt. args ist ein Array von String-Referenzen, das vom Java-System befüllt wird, bevor main() startet.

Beim Start mit:

C:\>java StringDemo stringA stringB stringC

Enthält args:
•	args[0] → "stringA"
•	args[1] → "stringB"
•	args[2] → "stringC"

Beispielprogramm:
[CODE]
public class StringDemo {
    public static void main(String[] args) {
        for (int j = 0; j < args.length; j++)
            System.out.println("Parameter " + j + ": " + args[j]);
    }
}
[/CODE]

7. Numerische Kommandozeilenargumente konvertieren

Kommandozeilenargumente sind immer Strings. Für Zahlen muss man konvertieren:

[CODE]
public class InputDemo {
    public static void main(String[] args) {
        int sum = 0;
        for (int j = 0; j < args.length; j++)
            sum += Integer.parseInt(args[j]);
        System.out.println("Sum: " + sum);
    }
}
[/CODE]

Bei Start mit C:\>java InputDemo 23 87 13 67 -42 ergibt sich Sum: 148.

Methode				Beschreibung

Integer.parseInt(String)	Wandelt String in int um
Double.parseDouble(String)	Wandelt String in double um

8. Argumente mit Leerzeichen

Enthält ein Argument Leerzeichen, einfach in Anführungszeichen einschließen:

C:\>java SomeProgram "one argument" "another argument"
Die Anzahl der Argumente ermittelt man mit args.length.

Zusammenfassung der Kernregeln

Thema				Wichtig
String[] arr = new String[n]	Erstellt n Zellen, alle null
arr[i] != null prüfen		Vor Zugriff auf Methoden eines Eintrags
arr.length			Anzahl Zellen, nicht Anzahl befüllter Zellen
String[] args in main()		Kommandozeilenargumente als String-Array
Integer.parseInt(args[j])	Umwandlung String → int



Kapitel 66 – Lineare Suche (Linear Search)

1. Lineare Suche – Grundprinzip

Die lineare Suche prüft jedes Element der Reihe nach, bis das gesuchte Element gefunden wird oder das Ende des Arrays erreicht ist.
Rückgabewert:

•	Gefunden → Index des Elements

•	Nicht gefunden → -1

2. Implementierung der Searcher-Klasse

[CODE]
class Searcher {
    // Sucht target im String-Array.
    // Gibt den Index zurück, oder -1 wenn nicht gefunden.
    public static int search(String[] array, String target) {
        for (int j = 0; j < array.length; j++) {
            if (array[j] != null)
                if (array[j].equals(target)) return j; // Gefunden!
        }
        return -1; // Nicht gefunden
    }
}
[/CODE]

Wichtig: Strings NIEMALS mit == vergleichen! Das vergleicht nur die Referenzen. Immer .equals() verwenden.

Die drei Fallen beim String-Vergleich:

Ausdruck		Ergebnis

array[j] = target	FALSCH – kopiert Referenz
array[j] == target	FALSCH – vergleicht Referenzen
array[j].equals(target)	RICHTIG – vergleicht Zeicheninhalt

3. Analyse von Algorithmen

Fall				Schritte
Bestes Ergebnis			1 Vergleich (Element steht an erster Stelle)
Schlechtestes Ergebnis		N Vergleiche (Element steht ganz hinten oder gar nicht vorhanden)
Durchschnitt			N/2 Vergleiche

4. Praktisches Beispiel: Telefonbuch

[CODE]
class PhoneEntry {
    private String name;
    private String phone;

    public PhoneEntry(String n, String p) { name = n; phone = p; }
    public String getName()  { return name; }
    public String getPhone() { return phone; }
}

class PhoneBook {
    private PhoneEntry[] phoneBook;

    public PhoneBook() {
        phoneBook = new PhoneEntry[5];
        phoneBook[0] = new PhoneEntry("James Barclay", "(418) 665-1223");
        phoneBook[1] = new PhoneEntry("Grace Dunbar",  "(860) 399-3044");
        phoneBook[3] = new PhoneEntry("Violet Smith",  "(312) 223-1937");
        phoneBook[4] = new PhoneEntry("John Wood",     "(913) 883-2874");
    }

    public PhoneEntry search(String targetName) {
        for (int j = 0; j < phoneBook.length; j++) {
            if (phoneBook[j] != null && phoneBook[j].getName().equals(targetName))
                return phoneBook[j]; // PhoneEntry-Objekt zurückgeben
        }
        return null; // Nicht gefunden
    }
}
[/CODE]

Ausdruck phoneBook[j].getName().equals(targetName) verstehen:

•	phoneBook[j] → Referenz auf PhoneEntry-Objekt

•	.getName() → liefert den name-String

•	.equals(targetName) → vergleicht die Strings

Wichtig: && ist ein Kurzschluss-Operator. Wenn phoneBook[j] == null ist false, wird .getName() nie aufgerufen → kein NullPointerException.


Kapitel 67 – Zweidimensionale Arrays (2D Arrays)

1. Das Konzept

Ein 2D-Array ist wie eine Tabelle (Zeile × Spalte). Jede Zelle wird mit zwei Indizes angesprochen: [Zeile][Spalte].

gradeTable[0][1]  // Zeile 0, Spalte 1
gradeTable[3][4]  // Zeile 3, Spalte 4

Indizes starten bei 0 – sowohl Zeilen als auch Spalten.

2. Deklaration und Konstruktion

[CODE]
int[][] myArray;                        // Referenzvariable (noch kein Array)
myArray = new int[3][5];               // 3 Zeilen, 5 Spalten, alle Werte = 0
Mit Initializer-Liste:
java
int[][] gradeTable = {
    {99, 42, 74, 83, 100},
    {90, 91, 72, 88,  95},
    {88, 61, 74, 89,  96},
    {61, 89, 82, 98,  93},
    {93, 73, 75, 78,  99},
    {50, 65, 92, 87,  94},
    {43, 98, 78, 56,  99}
};
[/CODE]

3. Bounds Checking

Zugriff auf eine nicht existierende Zelle führt zu:

ArrayIndexOutOfBoundsException
Array-Indizes müssen vom Typ int sein. gradeTable[3.5][2] ist ungültig.

4. Länge eines 2D-Arrays

Ausdruck		Bedeutung

myArray.length		Anzahl der Zeilen
myArray[0].length	Anzahl der Zellen in Zeile 0
myArray[row].length	Anzahl der Zellen in Zeile row

Jede Zeile kann eine andere Anzahl Zellen haben!

5. Unterschiedliche Zellenzahl pro Zeile

[CODE]
int[][] uneven = {
    {1, 9, 4},      // Zeile 0: 3 Zellen
    {0, 2},         // Zeile 1: 2 Zellen
    {0, 1, 2, 3, 4} // Zeile 2: 5 Zellen
};
uneven[1][2] → FEHLER! Zeile 1 hat nur 2 Zellen (Index 0 und 1).
[/CODE]

6. Implementierung (intern)

Ein 2D-Array ist ein Array von 1D-Arrays:

[CODE]
int[][] myArray;               // 1. Referenzvariable
myArray = new int[3][];        // 2. Erstellt Array mit 3 Zeilen-Referenzen (alle null)
myArray[0] = new int[3];       // 3. Zeile 0 erstellen
myArray[1] = new int[]{0, 2};  // 4. Zeile 1 mit Werten
Zeilen können einzeln durch neue 1D-Arrays ersetzt werden.
[/CODE]

7. Ausgabe eines 2D-Arrays (verschachtelte Schleifen)

[CODE]
for (int row = 0; row < uneven.length; row++) {
    System.out.print("Row " + row + ": ");
    if (uneven[row] == null) {
        System.out.print("empty");
    } else {
        for (int col = 0; col < uneven[row].length; col++)
            System.out.print(uneven[row][col] + " ");
    }
    System.out.println();
}
[/CODE]
Äußere Schleife: über Zeilen | Innere Schleife: über Spalten



Kapitel 68 – StringBuffer

1. String ist unveränderlich (immutable)

String-Objekte können nach ihrer Erstellung nicht verändert werden. Jede Änderung erzeugt ein neues Objekt:

String data = new String("Turtle");
data = data + 's'; // erstellt NEUES String-Objekt "Turtles"

Vorteil: Ein String in einer Methode bleibt garantiert unverändert, egal was andere Methoden tun.

2. charAt(int index) – Zeichen abfragen

String eg = "An Immutable String";
char ch0  = eg.charAt(0);            // 'A'
char ch1  = eg.charAt(1);            // 'n'
char last = eg.charAt(eg.length()-1); // 'g'

•	Indizes: 0 bis length()-1
•	Rückgabetyp: char

3. String umkehren – naive Methode (langsam)
[CODE]
public static String reverse(String data) {
    String rev = new String();
    for (int j = data.length()-1; j >= 0; j--)
        rev += data.charAt(j); // erstellt pro Zeichen ein neues Objekt!
    return rev;
}
[/CODE]

Problem: Pro Zeichen wird ein neues String-Objekt erzeugt → bei langen Strings sehr langsam.

4. Die Klasse StringBuffer

StringBuffer hält eine veränderliche Zeichenkette. Zeichen können eingefügt, gelöscht und ersetzt werden – ohne jedes Mal ein neues Objekt zu erstellen.

Konstruktoren:


new StringBuffer()             // leerer Buffer
new StringBuffer(int capacity) // Buffer mit Kapazität
new StringBuffer(String str)   // Buffer mit Startinhalt

Wichtige Methoden:

Methode				Beschreibung

append(char c)			Zeichen ans Ende anhängen
append(String s)		String anhängen
charAt(int i)			Zeichen am Index lesen
setCharAt(int i, char c)	Zeichen ersetzen
delete(int start, int end)	Zeichen löschen (start bis end-1)
insert(int index, char c)	Zeichen einfügen
reverse()			Reihenfolge umkehren
length()			Anzahl Zeichen
capacity()			Aktuelle Kapazität
toString()			StringBuffer → String umwandeln

5. String umkehren – effiziente Methode mit StringBuffer

[CODE]
public static String reverse(String data) {
    StringBuffer temp = new StringBuffer();
    for (int j = data.length()-1; j >= 0; j--)
        temp.append(data.charAt(j)); // kein neues Objekt pro Zeichen!
    return temp.toString();
}
[/CODE]

Nur 2 neue Objekte werden erstellt (statt eines pro Zeichen).

6. Palindrom-Detektor

Ein Palindrom liest sich vorwärts wie rückwärts, z.B. "A man, a plan, a canal, Panama!" (Leerzeichen und Groß-/Kleinschreibung ignoriert).

[CODE]
class Tester {
    public boolean test(String trial) {
        // 1. Alles in Kleinbuchstaben
        String lower = trial.toLowerCase();

        // 2. Nur Buchstaben 'a'-'z' in StringBuffer kopieren
        StringBuffer azBuffer = new StringBuffer();
        for (int j = 0; j < lower.length(); j++) {
            char c = lower.charAt(j);
            if (c >= 'a' && c <= 'z')
                azBuffer.append(c);
        }

        // 3. forward = original Reihenfolge, backward = umgekehrt
        String forward  = azBuffer.toString();
        String backward = azBuffer.reverse().toString();

        // 4. Vergleichen
        if (forward.equals(backward))
            return true;
        else
            return false;
    }
}
