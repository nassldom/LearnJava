<b>Part 11: Exceptions & Datei-E/A (Kapitel 100-106)</b>

In diesem Teil lernst du, wie Java mit Fehlern (Exceptions) umgeht und wie du Daten in Dateien speicherst oder daraus liest.

<b>1. Exceptions (Ausnahmen) (Kapitel 100-101)</b>
Eine Exception ist ein Ereignis, das den normalen Programmfluss unterbricht. Mit <code>try-catch-finally</code> kannst du darauf reagieren.

<b>Grundstruktur:</b>
[CODE]
try {
    // Code, der einen Fehler verursachen könnte
    int ergebnis = 10 / 0;
} catch (ArithmeticException e) {
    // Reaktion auf den Fehler
    System.out.println("Fehler: Division durch Null!");
} finally {
    // Wird IMMER ausgeführt (z.B. zum Schließen von Dateien)
    System.out.println("Aufräumarbeiten...");
}
[/CODE]

<b>Häufige Exception-Typen:</b>
- <code>ArithmeticException</code>: Mathematische Fehler (z.B. Division durch 0).
- <code>IndexOutOfBoundsException</code>: Zugriff auf ungültigen Array-Index.
- <code>NullPointerException</code>: Zugriff auf ein Objekt, das <code>null</code> ist.
- <code>InputMismatchException</code>: Falscher Datentyp bei der Eingabe (Scanner).

<b>2. Weiterwerfen und Eigene Exceptions (Kapitel 101)</b>
- <b>throws:</b> Deklariert in der Methodensignatur, dass eine Methode eine Exception auslösen könnte.
- <b>throw:</b> Löst manuell eine Exception aus.
- <b>Eigene Klasse:</b> Du kannst eigene Exception-Klassen erstellen, indem du von <code>Exception</code> erbst.

<b>3. Scanner, PrintWriter und Dateien (Kapitel 102, 106)</b>
Exceptions sind besonders wichtig beim Arbeiten mit Dateien (I/O), da hier oft unvorhersehbare Fehler auftreten (z.B. Datei nicht gefunden).

- <b>Scanner:</b> Zum Lesen von Textdateien.
- <b>PrintWriter:</b> Zum Schreiben in Textdateien.
- <b>File-Klasse:</b> Repräsentiert eine Datei oder ein Verzeichnis im Dateisystem. Mit ihr kannst du prüfen, ob eine Datei existiert (<code>exists()</code>), wie groß sie ist (<code>length()</code>) oder sie löschen (<code>delete()</code>).

<b>4. Binäre Dateien (Kapitel 103-105)</b>
Während Textdateien für Menschen lesbar sind, speichern binäre Dateien Daten im Format der Maschine.
- <b>DataOutputStream:</b> Zum Schreiben von primitiven Datentypen in binärer Form.
- <b>DataInputStream:</b> Zum Lesen von binären Daten.

<b>Merksatz:</b>
Nutze Exception Handling, um deine Programme robust gegen Benutzereingaben und externe Fehler (wie fehlende Dateien) zu machen.
