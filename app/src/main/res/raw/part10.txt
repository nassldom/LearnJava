PART 10 – Rekursion
(Kap. 90–96)

Grundprinzip:
• Methode ruft sich selbst auf
• Basisfall (Abbruchbedingung) ZWINGEND nötig!
• Sonst: StackOverflowError

Fakultät:
[CODE]
public static int fakultaet(int n) {
    if (n <= 1) return 1;        // Basisfall
    return n * fakultaet(n - 1); // rekursiver Aufruf
}
// fakultaet(5) = 5*4*3*2*1 = 120
[/CODE]

Fibonacci:
[CODE]
public static int fib(int n) {
    if (n <= 1) return n;        // Basisfall
    return fib(n-1) + fib(n-2); // rekursiver Aufruf
}
[/CODE]

String umkehren:
[CODE]
public static String umkehren(String s) {
    if (s.isEmpty()) return s;
    return umkehren(s.substring(1)) + s.charAt(0);
}
[/CODE]

Palindrom prüfen:
[CODE]
public static boolean istPalindrom(String s) {
    if (s.length() <= 1) return true;
    if (s.charAt(0) != s.charAt(s.length()-1)) return false;
    return istPalindrom(s.substring(1, s.length()-1));
}
[/CODE]

Fraktale (JavaFX):
• Schneeflocke, Sierpinski-Dreieck, Koch-Kurve
• Rekursion zeichnet immer kleinere Versionen desselben Musters
• Tiefe der Rekursion = Detailgrad

Merksatz:
• Jeder rekursive Aufruf muss das Problem verkleinern
• Basisfall ZWINGEND – sonst Endlosrekursion
• Rekursion ist elegant, aber langsamer als Schleifen
  (z.B. fib(50) dauert sehr lange ohne Optimierung)
