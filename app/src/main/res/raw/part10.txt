<b>Part 10: Rekursion (Kapitel 90-96)</b>

In diesem Teil lernst du das Prinzip der Rekursion kennen. Ein Problem wird gelöst, indem eine Methode sich selbst mit einer einfacheren Version des ursprünglichen Problems aufruft.

<b>1. Die zwei Teile der Rekursion (Kapitel 90)</b>
Jede rekursive Methode benötigt zwei essenzielle Bestandteile:
- <b>Basisfall (Base Case):</b> Eine einfache Situation, die ohne weiteren Selbstaufruf sofort gelöst werden kann.
- <b>Rekursiver Schritt:</b> Die Methode ruft sich selbst mit einem verkleinerten Problem auf.

<b>Hinweis:</b> Ohne einen erreichbaren Basisfall kommt es zur Endlos-Rekursion und einem <code>StackOverflowError</code>.

<b>2. Mathematische Beispiele (Kapitel 91-93)</b>
[CODE]
// Dreieckszahlen
public int triangle(int n) {
    if (n == 1) return 1; // Basisfall
    else return n + triangle(n - 1); // Rekursiver Schritt
}

// Fakultät (n!)
public int factorial(int n) {
    if (n == 0) return 1; // Basisfall
    else return n * factorial(n - 1);
}
[/CODE]

<b>3. Rekursion mit Strings und Grafik (Kapitel 94-96)</b>
- <b>Strings:</b> Rekursion eignet sich hervorragend, um Texte zu verarbeiten (z.B. Umkehren eines Strings).
- <b>Grafik & Fraktale:</b> Selbstähnliche Strukturen wie der Sierpinski-Teppich oder Fraktal-Bäume lassen sich elegant mit Rekursion zeichnen, indem eine Methode Teilfiguren von sich selbst zeichnet.

<b>Merksatz:</b>
Rekursion ist oft eleganter und kürzer als eine Schleife (Iteration), kann aber bei sehr vielen Aufrufen den Speicher (Stack) überlasten. Nutze Rekursion vor allem dann, wenn sich ein Problem natürlich in kleinere Teilprobleme zerlegen lässt.
