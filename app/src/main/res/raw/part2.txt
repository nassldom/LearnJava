<b>PART 2 – Erste Java-Programme</b>

<b>Kapitel 5 – Einführung in Java</b>

<i>Java</i> ist eine objektorientierte Programmiersprache, die besonders wegen ihrer <u>Plattformunabhängigkeit</u> beliebt ist. Das bedeutet: <b>Einmal kompilierter Java-Code läuft auf jedem Betriebssystem</b> – egal ob Windows, Linux oder macOS.

Das funktioniert so:

<b>1.</b> Du schreibst den Quellcode in einer <i>.java-Datei</i>.

<b>2.</b> Der Compiler (<i>javac</i>) übersetzt ihn in <i>Bytecode</i> (.class-Datei) – das ist kein direkter Maschinencode, sondern ein neutrales Zwischenformat.

<b>3.</b> Die <i>Java Virtual Machine (JVM)</i> interpretiert diesen Bytecode auf dem jeweiligen System und führt ihn aus.

<b>Codebeispiel (Hello.java):</b>

[CODE]
public class Hello {
    public static void main( String[] args ) {
        System.out.println("Hello World!");
    }
}
[/CODE]

<b>Erklärung zum Code:</b>

• <b>public class Hello</b> – Jedes Java-Programm besteht aus mindestens einer Klasse. Der Dateiname muss gleich dem Klassennamen sein: Hello.java.

• <b>public static void main(String[] args)</b> – Das ist der Einstiegspunkt. Die JVM startet hier.

• <b>System.out.println(...)</b> – Gibt Text auf der Konsole aus, mit Zeilenumbruch am Ende.

<b>Typische Fehler:</b>

• Klassenname und Dateiname stimmen nicht überein → Compilerfehler.

• Semikolon am Ende einer Zeile vergessen → Compilerfehler.

• Groß-/Kleinschreibung falsch (system statt System) → Compilerfehler.


<b>Kapitel 6 – Kleine Java-Programme</b>

Nachdem du „Hello World" kennst, geht es darum, den Aufbau weiterer Programme zu verstehen. Alle Java-Programme folgen derselben Grundstruktur: <b>Eine Klasse mit einer main-Methode</b>. Der Code innerhalb der Methode wird <u>Zeile für Zeile von oben nach unten</u> ausgeführt.

<b>Codebeispiel (HelloPlanets.java):</b>

[CODE]
public class HelloPlanets {
    public static void main( String[] args ) {
        System.out.println("Mercury");
        System.out.println("Venus");
        System.out.println("Earth");
        System.out.println("Mars");
    }
}
[/CODE]

<b>Erklärung zum Code:</b>

Jede println-Zeile gibt eine eigene Zeile auf der Konsole aus. Der Code wird strikt von oben nach unten ausgeführt. Die geschweifte Klammer <b>{</b> öffnet einen Block, <b>}</b> schließt ihn. Einrückung (4 Leerzeichen oder 1 Tab) macht den Code lesbar – das ist in der Ausbildung und in Teams <u>Pflicht</u>.

<b>Merksätze:</b>

• <b>System.out.println(...)</b> gibt Text mit Zeilenumbruch aus.

• <b>System.out.print(...)</b> gibt Text ohne Zeilenumbruch aus.

• Jede Anweisung endet mit einem Semikolon <b>(;)</b>.


<b>Kapitel 7 – Programme kompilieren und starten</b>

Bevor ein Java-Programm läuft, muss es <u>kompiliert</u> werden. Auf der Kommandozeile (Windows: Eingabeaufforderung / PowerShell) geht das so:

<b>Schritt 1 – Kompilieren:</b>

[CODE]
javac Hello.java
[/CODE]

Wenn kein Fehler auftritt, wird eine Datei <i>Hello.class</i> erzeugt (Bytecode).

<b>Schritt 2 – Ausführen:</b>

[CODE]
java Hello
[/CODE]

Die JVM startet die Klasse Hello und führt die main-Methode aus.

<b>Wichtige Regeln:</b>

• Der Dateiname muss genau dem Klassennamen entsprechen (Groß-/Kleinschreibung!).

• <i>javac</i> kompiliert → erzeugt .class-Datei.

• <i>java</i> führt aus → <u>kein .class am Ende angeben!</u>

• Die Datei muss im aktuellen Verzeichnis liegen, oder der Classpath muss stimmen.

<b>Typische Fehler:</b>

• <i>error: class Hello is public, should be declared in a file named Hello.java</i> → Dateiname falsch.

• <i>Could not find or load main class Hello</i> → Falsche Schreibweise beim Starten oder falscher Ordner.
